# Структура Kanban-дэшборда для СТО

## 1. Основная цель
Предоставить менеджеру наглядный интерфейс для управления ежедневными записями клиентов, позволяющий видеть весь день, вносить изменения и обрабатывать форс-мажорные ситуации.

## 2. Колонки (статусы)
1. **Новые заявки** — новые записи от клиентов через Telegram или вручную
2. **Подтвержденные** — клиент подтвердил запись, менеджер проверил
3. **В работе / Сегодня** — клиент ожидается на сервисе
4. **Выполнено** — запись выполнена, работа завершена
5. **Отменено / Перенесено** — отмены и переносы записей

## 3. Карточка клиента (каждая запись)
- Имя клиента
- Контакт (Telegram / телефон)
- Услуга
- Время визита
- Статус (колонка)
- Кнопки для действий:
  - Редактировать запись
  - Перенести запись
  - Отменить запись
  - Добавить комментарий

## 4. Поток данных
```
[Клиент] → [Telegram Bot] → [Backend] → [PostgreSQL] 
                                      │
                                      ├──→ [Kanban-дэшборд (менеджер)]
                                      │
                                      └──→ [Уведомления клиенту и сотруднику]
```
- Изменения на дэшборде автоматически синхронизируются с Telegram-ботом и базой данных
- Новые записи из Telegram появляются в Kanban автоматически

## 5. Особенности и функции
- Цветовое выделение карточек по статусу и срочности
- Возможность ручного добавления новых записей
- История изменений карточек (кто и когда изменил)
- Поддержка нескольких мастерских / филиалов
- Автосинхронизация с Telegram для уведомлений

## 6. Технологии MVP
- **Frontend:** React + библиотека Kanban (`react-beautiful-dnd`) или аналог
- **Backend:** FastAPI / Node.js
- **База:** PostgreSQL
