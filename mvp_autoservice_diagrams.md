# Диаграммы и интерфейсы MVP для СТО

## 1. Диаграмма потоков данных (MVP)
```
Клиент → Telegram Bot → Backend → PostgreSQL → Уведомление менеджеру
```
- **Клиент**: отправляет заявку (услуга, дата, время) через Telegram
- **Telegram Bot**: принимает запрос, отправляет в Backend
- **Backend (FastAPI / Node.js)**: обрабатывает данные, сохраняет в БД, формирует уведомления
- **PostgreSQL**: хранит историю клиентов и статусы заявок
- **Уведомление менеджеру**: Telegram / email / push уведомление о новой заявке или изменении статуса

## 2. Схема взаимодействия клиента с системой
1. Клиент впервые приходит в СТО
2. Менеджер информирует о Telegram-боте как способе быстрой записи
3. При повторном визите:
   - Клиент получает напоминание через SMS/email/Telegram
   - Переходит в Telegram-бот → выбирает услугу, дату, время
   - Получает мгновенное подтверждение заявки

## 3. Минимальный интерфейс Telegram-бота
- **Главное меню:**
  - Записаться на услугу
  - Мои записи
  - Информация о СТО

- **Процесс записи:**
  1. Выбор услуги
  2. Выбор даты и времени
  3. Подтверждение записи
  4. Получение уведомления о подтверждении

- **Уведомления:**
  - Подтверждение новой записи
  - Напоминание о предстоящем визите
  - Изменение статуса заявки (отменена / выполнена)

## 4. Визуальная схема MVP (текстовая версия)
```
[Клиент] ---> [Telegram Bot] ---> [Backend] ---> [PostgreSQL]
                                      |
                                      ---> [Уведомление менеджеру]
```
- В будущем можно расширить: подключение WhatsApp, веб-виджет, интеграция с CRM/1С
