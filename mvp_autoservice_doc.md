# Автоматизация записи клиентов для СТО

## 1. Общая информация
- **Название проекта:** Автоматизация записи клиентов для СТО
- **Цель:** Создать систему, которая позволяет клиентам записываться через Telegram, а мастерской — управлять заявками, уведомлениями и историей клиентов.
- **Проблема рынка:** 70–80% малых СТО в РФ не имеют CRM, работают по телефону/Excel, теряют клиентов.

## 2. Целевая аудитория
- Малые и средние СТО (1–5 мастерских)
- Владельцы и менеджеры, которые принимают звонки и хотят автоматизировать запись
- Ключевые сегменты:
  1. СТО общего профиля
  2. Детейлинг / премиальные сервисы
  3. Кузовной ремонт / шиномонтаж

## 3. Цели проекта
- Быстрый запуск MVP с реальной базой клиентов
- Проверка спроса и сбор первых KPI
- Подготовка к масштабированию и интеграции с внешними системами

## 4. Функциональные требования (MVP)
1. **Telegram Bot для записи**
   - Выбор услуги
   - Выбор даты и времени
   - Подтверждение заявки

2. **База данных (PostgreSQL)**
   - История клиентов
   - Статусы заявки: новая → подтверждена → выполнена → отменена

3. **Уведомления менеджеру**
   - Telegram / email
   - Уведомление о новой заявке и изменении статуса

4. **Импорт существующей клиентской базы**
   - Excel, CSV, 1С (выгрузка)
   - Автоматическое создание записей в системе

5. **Отчёты владельцу**
   - Количество новых заявок
   - Подтверждённые и выполненные
   - Повторные обращения

6. **Опционально:** бонус для клиента за использование Telegram (ускоренная запись или приоритет)

## 5. Нефункциональные требования
- Поддержка нескольких мастерских / филиалов
- Безопасность данных (хранение контактов, соответствие законам РФ о персональных данных)
- Docker для быстрой установки на VPS
- Возможность масштабирования (добавление каналов: WhatsApp, веб-виджет)

## 6. Архитектура решения
- **Клиент → Telegram Bot → Backend (FastAPI / Node.js) → PostgreSQL → уведомления менеджеру**
- Все данные централизованы в базе
- В будущем интеграция с 1С / Bitrix / Excel через webhook / API

## 7. Стратегия внедрения
1. **Сбор клиентской базы**: импорт всех существующих записей
2. **Настройка MVP бота**: запись + уведомления
3. **Обучение персонала**: как обрабатывать заявки
4. **Мотивация клиентов**: бонус или ускоренная запись
5. **Сбор KPI**: количество новых заявок, % клиентов использующих Telegram

## 8. План проекта (этапы)
| Этап | Что делаем | Срок |
|------|------------|------|
| 1 | Сбор и анализ клиентской базы | 1–2 дня |
| 2 | Разработка MVP (бот + база + уведомления) | 3–5 дней |
| 3 | Тестирование на 1–2 мастерских | 2 дня |
| 4 | Внедрение и обучение персонала | 1 день |
| 5 | Сбор обратной связи, доработка | 1–2 дня |
| 6 | Подготовка к масштабированию / интеграции | 2–3 дня |

## 9. Коммерческая стратегия
- **MVP**: внедрение + подключение базы → 60–70 тыс ₽
- **Поддержка**: 5–10 тыс ₽ / мес
- **Премиум пакет**: интеграция с 1С / Bitrix → доп. 20 тыс ₽

## 10. Риски и способы их минимизации
- **Пустая система без базы** → всегда подключаем существующую клиентскую базу
- **Сопротивление персонала** → обучение + простые инструкции
- **Пропуск заявок** → уведомления менеджеру и резервная панель

## 11. Диаграмма потоков данных (MVP)
```
Клиент → Telegram Bot → Backend → PostgreSQL → Уведомление менеджеру
```

## 12. Взаимодействие с клиентом
- Первичный контакт: телефон / визит
- Мотивация: бонус или ускоренная запись через Telegram
- Вторичное обращение: система отправляет напоминание → клиент использует Telegram
